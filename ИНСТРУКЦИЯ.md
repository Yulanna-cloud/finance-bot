# 🤖 Инструкция по запуску финансового бота «Герман»

Читай по шагам. Каждый шаг — отдельная задача. Не спеши.

---

## ШАГ 1. Получить токен Telegram-бота (5 минут)

1. Открой Telegram, найди бота **@BotFather**
2. Напиши ему `/newbot`
3. Он спросит имя бота — напиши любое, например `Мои финансы`
4. Спросит username — напиши латиницей с `bot` на конце, например `yulannafinance_bot`
5. BotFather пришлёт токен вида: `7123456789:AAHxxx...xxxyyy`
6. **Скопируй и сохрани этот токен** — он нужен на шаге 5 (переменная `TELEGRAM_BOT_TOKEN`)

---

## ШАГ 2. Получить Groq API ключ (5 минут)

Бот использует Groq (быстрый и бесплатный) для распознавания текста, голоса, фото чеков и выписок.

1. Открой в браузере: **https://console.groq.com/keys**
2. Войди или зарегистрируйся
3. Нажми **"Create API Key"**
4. Скопируй ключ вида `gsk_...` — сохрани его (переменная `GROQ_API_KEY`)

> ✅ Бесплатный тариф — щедрый лимит запросов в день, тебе хватит с запасом.

---

## ШАГ 3. Настроить доступ к Google Sheets (10 минут)

Это немного сложнее, но один раз и навсегда.

### 3.1 Создать Service Account

1. Открой: **https://console.cloud.google.com/**
2. Нажми **"Select a project"** → **"New Project"**
3. Назови проект, например `finance-bot` → нажми **"Create"**
4. В меню слева найди **"APIs & Services"** → **"Library"**
5. Найди **"Google Sheets API"** → нажми **"Enable"**
6. Найди **"Google Drive API"** → нажми **"Enable"**
7. Иди в **"APIs & Services"** → **"Credentials"**
8. Нажми **"+ Create Credentials"** → **"Service account"**
9. Имя: `finance-bot` → нажми **"Create and continue"** → **"Done"**
10. Нажми на созданный Service Account в списке
11. Перейди на вкладку **"Keys"** → **"Add key"** → **"Create new key"**
12. Выбери **JSON** → нажми **"Create"**
13. Скачается файл `finance-bot-xxx.json` — **не удаляй и не публикуй его!**

### 3.2 Дать боту доступ к твоей таблице

1. Открой скачанный JSON-файл любым текстовым редактором (Блокнот)
2. Найди строку `"client_email"` — там будет адрес вида `finance-bot@xxx.iam.gserviceaccount.com`
3. **Скопируй этот email**
4. Открой свою Google Таблицу
5. Нажми **"Поделиться"** (кнопка в правом верхнем углу)
6. Вставь скопированный email
7. Выбери роль **"Редактор"**
8. Нажми **"Отправить"**

> ✅ Теперь бот может писать в твою таблицу! Содержимое всего JSON-файла целиком идёт в переменную `GOOGLE_CREDENTIALS_JSON` (шаг 5).

## ШАГ 5. Запустить на Render (10 минут)

Render бесплатен навсегда и не требует карты, но бесплатный сервис "засыпает"
после 15 минут без входящих запросов. Поэтому у бота есть встроенный
маленький веб-сервер (health-check) — а в Шаге 5.3 мы настроим бесплатный
внешний "будильник", который будет пинговать его и не давать заснуть.

### 5.1 Создать сервис

1. Открой **https://render.com**, зарегистрируйся через GitHub (карта не нужна)
2. Нажми **"New +"** → **"Web Service"**
3. Выбери репозиторий `finance-bot`
4. Render сам увидит `Dockerfile` в репозитории и предложит **Docker** окружение — соглашайся
5. Instance Type — **Free**
6. Нажми **"Create Web Service"**, Render начнёт сборку (пару минут)

### 5.2 Добавить переменные окружения (очень важно!)

Во вкладке **"Environment"** добавь:

**Переменная 1 (обязательно):**
```
Name:  TELEGRAM_BOT_TOKEN
Value: (токен из Шага 1)
```

**Переменная 2 (обязательно):**
```
Name:  GROQ_API_KEY
Value: (ключ из Шага 2)
```

**Переменная 3 (обязательно):**
```
Name:  GOOGLE_CREDENTIALS_JSON
Value: (открой JSON-файл из Шага 3 в Блокноте, выдели ВЕСЬ текст и вставь)
```

**Переменная 4 (опционально — ежемесячные напоминания):**
```
Name:  OWNER_CHAT_ID
Value: (твой chat_id в Telegram, например через @userinfobot)
```

**Переменная 5 (опционально — чтение QR-кода на чеке через ФНС):**
```
Name:  PROVERKACHEKA_TOKEN
Value: (токен сервиса proverkacheka.com, если пользуешься)
```

После сохранения переменных Render сам перезапустит бота. Во вкладке
**"Logs"** должно появиться: `Бот запущен!`.

Скопируй публичный адрес сервиса сверху страницы — он вида
`https://finance-bot-xxxx.onrender.com`. Он понадобится в следующем шаге.

### 5.3 Настроить "будильник", чтобы бот не засыпал

1. Открой **https://cron-job.org**, зарегистрируйся (бесплатно, без карты)
2. **"Create cronjob"**
3. URL — вставь адрес из Шага 5.2 (просто сам адрес, без ничего после)
4. Интервал — каждые **10 минут**
5. Сохрани

> ✅ Теперь внешний сервис каждые 10 минут "стучится" в бота, Render видит
> входящий запрос и не усыпляет сервис — бот работает круглосуточно.
> Уложится в бесплатный лимит Render (750 часов/мес — как раз на весь месяц).

**Переменная 1 (обязательно):**
```
Name:  TELEGRAM_BOT_TOKEN
Value: (токен из Шага 1)
```

**Переменная 2 (обязательно):**
```
Name:  GROQ_API_KEY
Value: (ключ из Шага 2)
```

**Переменная 3 (обязательно):**
```
Name:  GOOGLE_CREDENTIALS_JSON
Value: (открой JSON-файл из Шага 3 в Блокноте, выдели ВЕСЬ текст и вставь)
```

**Переменная 4 (опционально — ежемесячные напоминания):**
```
Name:  OWNER_CHAT_ID
Value: (твой chat_id в Telegram, например через @userinfobot)
```
28-го числа бот напомнит посмотреть отчёт, 1-го — спланировать бюджет на новый месяц.

**Переменная 5 (опционально — чтение QR-кода на чеке через ФНС):**
```
Name:  PROVERKACHEKA_TOKEN
Value: (токен сервиса proverkacheka.com, если пользуешься)
```

8. После добавления переменных Railway автоматически перезапустит бота
9. Во вкладке **"Logs"** должно появиться: `Бот запущен!`

## ШАГ 6. Проверить что всё работает

1. Найди своего бота в Telegram по username из Шага 1
2. Напиши `/start` — он должен ответить приветствием и показать кнопку **☰ Меню**
3. Напиши текстом: _«кофе 350»_ или голосовым: _«потратила на кофе триста рублей»_
4. Проверь свою таблицу — в листе ОПЕРАЦИИ должна появиться новая строка!

## Как пользоваться

### Текст 💬
Пиши как обычно: «такси 300», «Пятёрочка 1200», «Вакуматор на ВБ 1545».

### Голосовые сообщения 🎤
- «купила воду за двадцать рублей»
- «потратила в Пятерочке тысячу четыреста»

### Фото чека 📷
Сфотографируй чек (нужен хороший свет) — бот прочитает все позиции и запишет каждую отдельно.
Фото с подписью («обучение Рите танцы») — бот использует подпись вместо автоопределения.

### Банковская выписка 📄
Отправь файл (PDF/CSV/Excel) — бот прочитает и запишет все операции.

### Кнопка ☰ Меню
Открывает: отчёт за месяц, итоги года, анализ трат, планирование, бюджет, удаление/восстановление записи, изменение записи, архив, помощь.

### Команды
`/start`, `/otchet`, `/year`, `/analiz`, `/edit`, `/budget`, `/plan`, `/delete`, `/restore`, `/archive`, `/fix`, `/pomosh`

## Если что-то не работает

**Бот не отвечает** → проверь вкладку Logs на Render, там будет написано что случилось. Если давно не было пингов — подожди 30-60 секунд, сервис "просыпается"

**Бот отвечает с задержкой в первый раз** → нормально, если будильник (Шаг 5.3) настроен недавно или интервал пинга больше 15 минут — сервис успел заснуть и сейчас "просыпается"

**Ошибка Google Sheets** → убедись что дал доступ service account (Шаг 3.2)

**Неверно распознан текст/голос/фото** → проверь `GROQ_API_KEY` в Variables, посмотри Logs

**Голос не расшифровывается** → попробуй сказать текстом
